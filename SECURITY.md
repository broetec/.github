# Security policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in a **Broetec** project, please report it **privately** so we can assess and address it before public disclosure.

**Preferred:** use [GitHub Security Advisories](https://docs.github.com/en/code-security/security-advisories) for the affected repository if that feature is enabled (tab **Security** → **Report a vulnerability**).

**Email:** if private reporting is not available for the repo, write to **contato@broetec.com.br** with:

- A short description of the issue and its impact
- Steps to reproduce (or a proof of concept), if possible
- Affected versions or commit, if known

Please allow a reasonable time for triage and a fix before discussing the issue in public forums.

## Scope

Each Broetec repository may define its own supported versions and release process. This file sets **organization-wide** expectations for how to reach us; project maintainers may add a `SECURITY.md` in their repo with product-specific details.

## Safe harbor

We support responsible disclosure. If you act in good faith—avoiding privacy violations, destruction of data, or disruption of services—we will not pursue legal action for research conducted under this policy.
